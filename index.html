Quick Guideline

Installation

1. Go to page : https://nodejs.org/en and download a version of NodeJS that has the word LTS (a.k.a. Long Term Support), for example : version 14.15.0 LTS.

2. Install NodeJS.

Double click "install.bat". It can take about 5-10 minutes for the first time you install this tool.

Double click "open_Cypress.bat". It can take a few minutes to open the tool.



Starting


Implemented :

  1. Support changing config between different test environments.
  2. API is simple & easy to use if you only need basic test cases.
  3. Implementation of a simple text UI in the Console window of the web browser, for viewing a detailed bug report after completing each test case.

  4. Support setting a specific test account for a test suite (it will be reset back to the default test account for other test suites).
  5. Support re-testing failed test cases based on their IDs printed at the end of the log in the Console window of the web browser.
  6. Support re-testing test cases starting from a specific test case ID.
  7. Support re-testing specific test suites.
  8. Support importing multiple test files into one test file and still support re-testing features.

  9. Support declaring sample inputs of Intents, Entities to simplify input management in test files. Input can be specific or random.
  10. Support sending value and action of a button to VA instead of sending a message.
  11. Comparing messages is more precise in case the response message contains dynamic values.
  12. Support comparing random response messages returned by VA.
  13. Support comparing buttons in the response message to the expected buttons.
  14. For technical limitations, the tool cannot receive Timeout messages but still support you to check the main logic of flow Timeout by using the wait feature of Cypress.

  15. Support checking that no Incident or Call Record should be logged after a step.
  16. Support checking information of a new Incident or Call Record after a step.
  17. Support checking information of an old Incident or Call Record.
  18. Support checking information of an anonymous Incident that you don't know its ticket number but know what is changed in the ticket.
  19. Support checking chat transcript. This feature is not perfect but can be useful.
  20. Support checking Hand-Off connection from client-side to server-side (not vice versa) after VA returns the connection message.
  21. Auto close Hand-Off connection on ServiceNow after checking.

  22. Support preparing data : set state of an Incident to an expected state.
  23. Support preparing data : manipulate the state of multiple Incidents so that the total number of Incidents with the expected state should be <= expected quantity.

  24. Support generating manual test cases in Excel format with no test report.
  25. Support generating manual test cases in Excel format with the bugs inserted as comments to corresponding steps.
